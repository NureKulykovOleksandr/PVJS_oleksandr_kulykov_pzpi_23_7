## 📂 Структура репозиторію

Репозиторій розділений на 4 папки, кожна з яких представляє собою окремий працюючий вебзастосунок на базі **Vue 3 + Vite**:

### 1. [lab1](./lab1) — Керування списком задач (To-Do List)
*   **Опис:** Простий і лаконічний менеджер задач.
*   **Технології:** Vue 3, Composition API, реактивні змінні (`ref`), двостороннє зв'язування (`v-model`), директиви рендерингу (`v-for`, `v-if`), computed-фільтрація задач.

### 2. [lab2](./lab2) — Галерея зображень Picsum API
*   **Опис:** Асинхронна галерея картинок із підвантаженням з мережі.
*   **Технології:** Vue 3, Picsum Photos API (асинхронні запити `fetch` з `async/await`), скелетони завантаження (skeleton screens), обробка станів завантаження та помилок (з можливістю перезапуску), клієнтська пагінація та збереження обраних картинок у `localStorage`.

### 3. [lab3](./lab3) — Менеджер задач TaskFlow (Pinia & Router)
*   **Опис:** Повноцінний та розширений планувальник завдань із глобальним станом та маршрутизацією.
*   **Технології:** Vue 3, Pinia (управління станом з автоматичною синхронізацією в `localStorage`), Vue Router (5 маршрутів: список, деталі, створення, редагування, статистика), плавні переходи між сторінками (`<Transition>`), валідація форм дедлайнів, сторінка статистики (виконана виключно на геттерах сховища).

### 4. [lab4](./lab4) — Портал подій EventHub (Composition API & Composables)
*   **Опис:** Інформаційний портал подій та реєстрації з оптимістичним інтерфейсом користувача.
*   **Технології:** Vue 3 Composition API, користувацькі composables (`useLocalStorage` для синхронізації, `useEvents` для подій, `useRegistration` для реєстрації), **Optimistic UI Updates** (миттєве додавання учасника у список з імітацією 1.5с затримки сервера та 20% шансом помилки мережі, що викликає відкат стану – rollback), `<Teleport>` для модальних вікон у body, `<KeepAlive>` для кешування стану сторінок при навігації.

## 🚀 Як запустити локально будь-яку лабораторну роботу

1.  Встановіть [Node.js](https://nodejs.org/) (рекомендовано версію LTS).
2.  Склонуйте цей репозиторій:
    ```bash
    git clone https://github.com/NureKulykovOleksandr/PVJS_oleksandr_kulykov_pzpi_23_7.git
    ```
3.  Перейдіть до папки потрібної лабораторної роботи (наприклад, `lab3`):
    ```bash
    cd lab3
    ```
4.  Встановіть залежності:
    ```bash
    npm install
    ```
5.  Запустіть локальний сервер для розробки:
    ```bash
    npm run dev
    ```
6.  Відкрийте посилання у браузері (зазвичай `http://localhost:5173`).
